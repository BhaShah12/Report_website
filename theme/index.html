Motivation

Information retrieval is a critical component of crisis response, but it's not always easy to find reliable sources in the midst of chaos. Rumors, gossip, and wild speculation run rampant, leaving authorities scrambling to sort fact from fiction. In this scenario, there is a need for a global, reliable and user friendly application to provide quick and accurate facts to the general masses.

Dataset

CMGE uses the stream of data from social media (Twitter, Reddit), News about an event based on which we can ask queries to the system and get answers based on the news content.
The CrisisFACTs dataset is a collection of tweets that were posted during various crisis events, including natural disasters, terrorist attacks, and other emergencies. The dataset was created to help researchers and developers better understand how people use social media during times of crisis and to develop new tools and technologies to support crisis response and recovery efforts.

Analyzing these data streams can provide a more complete picture of the crisis event, including how people are responding, what information they are sharing, and what actions are being taken by authorities and organizations.

Approach

  • The first step in this task is to collect and preprocess the data. This involves gathering a stream of data from various sources such as social media (Twitter, Reddit), news articles, and other relevant sources.
  • The data is then preprocessed, which includes tasks such as tokenization, part-of-speech tagging, and named entity recognition, to extract relevant information from the text.

  • Once the data is preprocessed, the next step is to build a search index. The search index is a data structure that enables efficient searching and retrieval of documents based on their content. The search index is built using techniques such as inverted indexing and vector space modeling.
  • The next step is to build a query interface that allows users to ask questions about the data. The query interface can take various forms, such as a search engine, a chatbot, or a voice assistant. The interface should be able to process natural language queries and return relevant answers based on the content of the documents.
  • When a user submits a query, the query interface will process the query and use the search index to retrieve relevant documents. The retrieved documents are then ranked based on their relevance to the query, and the top-ranked documents are presented to the user as the answer.
  • Finally, the answer is presented to the user in a suitable format, such as a summary, a list of relevant documents, or a dialogue with a chatbot.
  • Overall, the goal of this task is to enable users to retrieve relevant information from a stream of data based on their queries, and to present the information in a user-friendly and informative way.
